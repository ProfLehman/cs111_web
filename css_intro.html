Introduction to CSS

Cascading Style Sheets (CSS) are the recommended way to format your HTML web pages.

HTML specifies the structure

CSS specifies the formatting

It is considered best practice to separate the structure and formatting.

For an introduction and reference to CSS visit W3 Schools.

Rules and Selectors

CSS consists of a set of formatting rules applied to elements via selectors. Each rule specifies a property and value. Multiple rules can be applied or cascaded. The selector describes the element for which the rule will be applied.

In the following example, the paragraph text is set to the color green with font size of 18pt. In this example the selector is for all paragraph tags, so the rules are applied to all paragraphs.

p {
 color:green;
 font-size:18pt;
}

This is a sample paragraph with green 18pt text.

Select By Element

Styles can be applied to all elements by specifying the name of the element and the style properties. 

In this example all paragraph elements <p> will be formatted with "Times New Roman" font, font size 14 points, and have the first line of text indented .25".  If "Times New Roman" font is not available, the "Times" font will be used.  If "Times" is not available a default "serif" font will be used.

The style definition for all <p> tags goes in the CSS style sheet.

p {
  font-family: "Times New Roman", Times, serif;
  font-size: 14pt;
  text-indent: .25in;
}

Inline CSS

CSS styles can be defined for each element by adding the style attribute to the tag.

Styles defined this way only apply to a single element. This approach is OK for small web pages or occasional use, but is difficult to maintain for larger sites. NOTE: Do not use Inline CSS for the CS111 Web Assignment

In this example the background color for the paragraph is set to orange and the font is set to Arial.

<p style="color:red; font-family:Arial;">
This is a sample paragraph with red text using Arial font</p>

This is a sample paragraph with red text using Arial font.

Internal CSS

CSS styles can be defined in the <head> section of your web page. Selectors are placed withing the <style> tag.

Styles defined this way apply to the entire page. While this approach is OK for single web pages, it is not a good approach when styles need to be applied to more than one page. NOTE: Do not use Internal CSS for the CS111 Web Assignment

In this example the background color for the page would be set to white ie. #FFFFFF using the RGB hexadecimal color model. The text color for all heading 1 tags would be green.

<head>

  <style>

    body {

      background-color: #FFFFFF;

    }

    h1 {
      color: #00FF00;
    }

  </style>

</head>

External CSS

CSS styles can also be defined as a separate file. Defining the CSS styles in a separate file allows the styles to be easily applied to multiple pages.

A link must be added to your web page to use an external style sheet. The following example shows a link to a CSS sheet called style.css.

<head>

<link rel="stylesheet" type="text/css" href="style.css" />

</head>

Sample CSS file style.css

The following CSS file demonstrates several selectors and common CSS formatting rules.

Selectors

The h1 selector specifies the rules that will be applied to all h1 tags ie. main or largest heading size for entire page.

The p selector specifies the rules that will be applied to all p tags ie. paragraphs.

The ol selector specifies the rules that will be applied to all ol tags ie. ordered lists.

The body selector specifies the rules that will be applied to all of the content within the body tags ie. most of the page.

Rules

font-family specifies the font to be used.

list-style specifies the numbering for the ordered list.

color specifies the text color. Color can be specified with color names or hex codes.

background-color specifies the background text color. Background-color can be specified with color names or hex codes.

/*
style.css
demonstrates basic CSS rules
*/

h1 {
 font-family: "Comic Sans MS", Arial, sans-serif;
 color: orange;
}

p {
 color:green;
 font-size:18pt;
}
ol {
 list-style: upper-roman;
}

body {
 background-color: #F0F0F0;
}

(Optional) Select By Element ID

Styles can be applied to single element instances by declaring an id.  An id must be unique to a specific element instance.  The # sign is used to denote the style for a specific id.

The style definition for the h3 tag identified by the id "special" is added to the CSS style sheet.

h3#special {
  color : red;
}

The following code is placed in the <body> section. 

In this example, the second heading is given the id attribute "special".  The <h3> heading with the id of "special" will have the formatting specified by the style.  In this case the special style specifies the color red.  The <h3> heading without the id will have the default formatting.

(Optional) Select By Element Class

Styles can be applied to multiple element instances by defining assigning a class name to elements.  The same class name can be used for all elements to which the style is applied.

The style definition for an ordered list with the class name "roman" is added to the CSS style sheet.

ol.roman{
   list-style :upper-roman;
}

In this example, the first and third ordered lists will be formatted using the style defined above.  The second list will be formatted with the default style.

<ol class="roman">
   <li>one</li>
   <li>two</li>
   <li>three</li>
</ol>

<ol>
   <li>four</li>
   <li>five</li>
   <li>six</li>
</ol>

<ol class="roman">
   <li>seven</li>
   <li>eight</li>
   <li>nine</li>
</ol> 

In this example all elements of class "roman" are formatted as upper-case roman numerals.

(Optional)Validating CSS

To ensure your CSS is correct you should "Validate" or "check" your CSS at http://jigsaw.w3.org/css-validator/.

Validating your CSS will help to ensure all browsers will render your pages correctly and efficiently. Your CSS code should not have any warnings or errors. Fix the first error (or two) and then retry the validator. The number of errors reported is not accurate.
